Boost QVM

View Proxies

Boost QVM defines various function templates which take a reference to a quaternion/vector/matrix object and return an object of unspecified static quaternion/vector/matrix type for which the quat_traits/vec_traits/mat_traits template is specialized to provide a different (possibly mutable) view to the original quaternion/vector/matrix object.

For example, given a vector v, the expression col_mat(v) returns a view proxy that makes v look and feel like a matrix-column. More complex views are created by simple composition, for example transposed(col_mat(v)) is equivalent to row_mat(v).

Note: typically, the returned view proxies are simply the original object reference reinterpret_casted to an internal quaternion/vector/matrix type. This technique avoids creating a temporary object (these internal types are made non-copyable to prevent potential undefined behavior, however they do define operator= that acts on the original object.)
