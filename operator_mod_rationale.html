Boost QVM

Rationale for overloading operator , (comma)

Boost QVM overloads operator, to implement swizzling. Suppose we have a 3D vector v3 and a 2D vector v2. Obviously, if we wanted to multiply v3 by a scalar, we'd use:

v3 *= 42;

By overloading operator, Boost QVM makes the following expressions legal as well:

(v3,Y) *= 42; //Multiply only the Y component of v3 by 42.
(v3,YZ) *= 42; //Multiply only the Y and Z components of v3 by 42.
v2 = (v3,ZX); //Assign v3's Z component to v2's X component, and v3's X component to v2's Y component.

Operator, was chosen for its visual appeal and because other operators with more appropriate precedence (for example, operator() or operator[]) are required to be member functions. Since Boost QVM is designed to work with user-defined quaternion, vector and matrix types, it can not provide generic definitions of those operators.

(Thanks to Michael Marcin for the idea to use operator, for swizzling.)
